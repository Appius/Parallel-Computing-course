Про этот курс

В курсе рассматриваются основы параллельного программирования на языке программирования Java. История возникновения параллельных программ, основные примитивы на языке Java, конструкции для синхронизации работы потоков и механизмы общения.

Материал курса состоит в основном из видео-лекций. К каждой части прилагаются слайды лекции с возможностью скачивания, а также демо - пример работы программы с использованием выученного материала, код выложен на GitHub. После каждого юнита (части лекции) есть несколько (от 1 до 5) тестов. Результаты тестов не будут учитываться при подсчете финальной оценки; она будет состоять из оценок за домашние задания и финальные тесты к лекциям. После каждой лекции есть домашнее задание, которое необходимо сделать и загрузить в виде архива на сервер. Крайний срок сдачи домашнего задания или прохождения финального теста - 2 недели с момента публикации. Рабочий язык лекций - русский.

Программа курса

Лекция 1: Introduction to Parallel Computing

В первой лекции рассматриваются основные виды параллелизма, причины появления и необходимости использования, способы написания высокопроизводительных приложений. Закон Амдала как один из основных законов, описывающий ограничения потенциального прироста производительности, получаемой за счет распараллеливания программ. Декомпозиция задач и конвейерный метод. В конце рассматривается поддержка параллелизма в Java, concurrency.

Лекция 2: Synchronization and threads interaction

Вторая лекция посвящена проблемам взаимодействия потоков в многопоточной программе. Рассмотрены основные конструкции для синхронизации и взаимной работы потоков. Проблемы взаимной блокировки, live lock, resource starvation. Шаблон проектирования Producer-Consumer и его применение для многопоточного приложения. java.util.concurrent.

Лекция 3: Producer-Consumer model

В третьей лекции рассматривается модель Producer-Consumer: разные режимы работы, подбор параметров системы для оптимальной работы, готовая реализация на языке Java: ThreadPoolExecutor.

Лекция 4: Actors Model of Computation

В четвертой лекции рассказывается про модель акторов, как возможной схемы взаимодействия потоков. Akka. Основные принципы работы, демо написания простого приложения на Аkka. Проблема обедающих философов как классический пример, иллюстрации проблемы синхронизации потоков и разных вариантов ее решения.

Лекция 5: Software Transactional Memory

Заключительная лекция посвящена STM - программная транзакционная память - механизму управления параллелизмом.

Требования

Данный курс требует базовых знаний Java и представлений о работе многопоточных приложений. Английский - базовый уровень.
