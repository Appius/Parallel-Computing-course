В каждой из лабораторных работ необходимо реализовать модель producer-consumer, используя стандартные Java API. Необходимо провести вычисления и собрать статистику о том, как количество потоков и длительность транзакции влияет на время обработки всего файла. Для имитации задержки на транзакцию можно использовать Thread.sleep(N). Условия задач даны ниже. Все транзакции описываются в виде бинарного файла, который состоит из int'ов (4 байта). 

Вариант 1. Перечисление средств 
Дан набор счетов, которые проиндексированы от 0 до N-1, где N - количество счетов. Между счетами происходят перевод денег (каждый перевод описан в файле). Необходимо посчитать баланс каждого счёта после того, как все операции выполнены. Баланс на счёте может "уходить в минус" - то есть, допустимо снимать со счёта больше средств чем на нём есть. 
Структура файла. Каждое значение - int (4 байта)
<общее количество счетов><сумма на счету #0><сумма на счету #1><сумма на счету #N-1>
<общее количество переводов><перевод 1><перевод 2><перевод 3><перевод K>
Где перевод описывается тремя int'ами - <со счёта><на счёт><сумма>

Вариант 2. Социальная сеть
Дан граф социальной сети (ненаправленный граф). Ноды графа - пользователи, а рёбра - отношение "дружбы" между ними. Ноды графа проиндексированы последовательно, от 0 до N-1, где N - количество нод. Ребро описывается как пара нод. Пользователи получают "голоса". При этом - сам пользователь получает 100% "голосов", а его "друзья" - 50% каждый. 
<Количество Нод (N)><Количество рёбер>
<Ребро 0><Ребро 1>....<Ребро N-1>
<Количество голосований><Голосование 1><Голосование 2>...<Голосование K> 
Где голосование описывается двумя int'ами - <номер ноды><количество голосов> 

Вариант 3. GSM Вышки
Даны координаты GSM вышек которые обслуживают вызовы. У каждой вышки есть радиус действия в пределах которого вышка может обслуживать вызов. Посчтитать, сколко вызовов попало в зону действия каждой из вышки. 
<Количество Вышек (N)>
<Вышка 0><Вышка 1>....<Вышка N-1>
<Количество вызовов><Вызов 0><Вызов 1>...<Вызов K>
Вышка описывается тремя int'ами - <x, y, радиус действия> 
А вызов - двумя int'ами - <x, y>

Задание

Задание загружать через форму ниже. Формат должен быть следующим: архив .zip, для каждого задания отдельная папка, в каждой из которых содержится список файлов .java, один из которых содержит public static void main(String[] args).
